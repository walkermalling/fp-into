  1. a pure function is a mathematical function

2.3.4 What is a mathematical function?

  1. just a relation between two values
  2. "Functions can be described as a set of pairs with the position (input, output)
  3. you can model a function by a table, and the Javascript notation myFunction[input] would be equivalent to myFunction(input), where in the first myFunction is an object of key-value pairs and in the second it is actually a function that implements the mathematical relation between input and output.

2.3.5 Practicalities of Purity

  1. function output can always be cached based on input
  2. portable (everything you need to know is in its argument and its internals)
  3. testable (since you're forced to inject dependencies, you can mock them easily)
  4. referential transparency

    "A spot of code is referentially transparent when it can be substituted for its evaluated value without changing the behavior of the program."

  5. parallelization

    "we can run any pure function in parallel since it does not need access to shared memory and it cannot, by definition, have a race condition due to some side effect"

2.3.6 Making Pure Functions

  1. sometimes you can conver impure function into pure function by "delaying execution"

    this really just means separating the concept of an action from its execution; the action is a value and can be passed around as a function; the execution or procedure is its side effect
